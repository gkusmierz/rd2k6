# CAE-003: Audio Recording Engine

## Kontekst biznesowy

Nagrywanie audio jest kluczowe dla stacji radiowych -- nagrywanie voicetrackow, zapisy na zywo, automatyczne nagrywanie z harmonogramu. System obsluguje nagrywanie z roznym kodowaniem (PCM, Ogg, MP2), z kontrola kanalow i sample rate, oraz z funkcja VOX (voice-activated recording) ktora rozpoczyna faktyczne nagrywanie dopiero po wykryciu sygnalu powyzej progu.

## Aktorzy

| Aktor | Rola w tej feature |
|-------|-------------------|
| Klient TCP (rdcatch, rdlibrary) | Steruje nagrywaniem (load, record, stop, unload) |
| Audio Driver | Wykonuje fizyczne nagrywanie z hardware |

## Granica funkcjonalnosci

```
IN SCOPE:
  - Ladowanie nagrywania z parametrami kodowania (LR)
  - Rozpoczecie nagrywania z max dlugoscia i progiem VOX (RD)
  - Zatrzymanie nagrywania (SR)
  - Zwolnienie nagrywania (UR)
  - Ownership tracking (record_owner per stream)
  - Asynchroniczne notyfikacje (stateRecordUpdate)
  - VOX (voice-activated recording start)

OUT OF SCOPE:
  - Odtwarzanie → patrz CAE-002
  - Kontrola glosnosci wejscia → patrz CAE-004
  - Metering → patrz CAE-005
```

## Reguly biznesowe (Gherkin)

```gherkin
Rule: Nagrywanie z VOX

  Scenario: Nagrywanie z progiem VOX
    Given recording zaladowany
    When  RD card stream length threshold
    Then  record_length i record_threshold zapisane
    And   record_owner[card][stream] = client_id
    And   nagrywanie czeka na sygnal > threshold
    And   stateRecordUpdate(0=Recording) → "RD ... +!"

  Scenario: VOX aktywacja
    Given nagrywanie w trybie czekania
    When  poziom audio przekracza threshold
    Then  stateRecordUpdate(4=RecordStarted) → "RS card stream +!"
    And   faktyczne nagrywanie do pliku rozpoczete

  Scenario: Zatrzymanie nagrywania
    Given nagrywanie aktywne
    When  SR card stream
    Then  stateRecordUpdate(3=Stopped) → "SR card stream +!"
  # Zrodlo: cae.cpp:840-925, 1481-1506 | Pewnosc: potwierdzone

Rule: Parametry kodowania

  Scenario: Ladowanie z kodowaniem
    Given card < RD_MAX_CARDS, port < RD_MAX_PORTS
    And   coding < 5, channels <= 2
    When  LR card port coding chans srate brate filename
    Then  driver laduje nagrywanie z podanymi parametrami
    And   "LR ... stream +!"

  Scenario: Unload zwraca dlugosc

    When  UR card stream
    Then  driver zwraca dlugosc nagrania
    And   "UR card stream length +!"
  # Zrodlo: cae.cpp:794-836 | Pewnosc: potwierdzone
```

## API klas w scope

### MainObject (recording slots)

**Sloty nagrywania:**
| Slot | Parametry | Efekt |
|------|-----------|-------|
| loadRecordingData | int id, unsigned card, unsigned port, unsigned coding, unsigned channels, unsigned samprate, unsigned bitrate, QString name | Laduje nagrywanie |
| unloadRecordingData | int id, unsigned card, unsigned stream | Zwalnia nagrywanie, zwraca dlugosc |
| recordData | int id, unsigned card, unsigned stream, unsigned len, int threshold | Rozpoczyna nagrywanie |
| stopRecordingData | int id, unsigned card, unsigned stream | Zatrzymuje nagrywanie |
| stateRecordUpdate | int card, int stream, int state | Callback: 0=Recording, 2=Paused, 3=Stopped, 4=Started |

**Driver Interface (recording):**
| Metoda | Zwraca | Opis |
|--------|--------|------|
| *LoadRecord(card, port, coding, chans, samprate, bitrate, wavename) | bool | Laduje nagrywanie |
| *UnloadRecord(card, stream, *len) | bool | Zwalnia, zwraca dlugosc |
| *Record(card, stream, length, thres) | bool | Rozpoczyna |
| *StopRecord(card, stream) | bool | Zatrzymuje |

## Protokoly komunikacji

| Komenda | Parametry | Odpowiedz | Znaczenie |
|---------|-----------|-----------|-----------|
| LR | card port coding chans srate brate filename | LR ... stream +! | Load Recording |
| UR | card stream | UR card stream length +! | Unload Recording |
| RD | card stream length threshold | RD ... +! | Record |
| SR | card stream | SR card stream +! | Stop Recording |

Asynchroniczne: RS card stream +! (VOX record started)
