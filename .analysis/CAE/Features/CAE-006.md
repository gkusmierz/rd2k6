# CAE-006: Daemon Lifecycle, Provisioning & Codec Detection

## Kontekst biznesowy

Zanim daemon CAE moze obslugiwac klientow, musi przejsc sekwencje inicjalizacji: wczytanie konfiguracji, polaczenie z baza danych, auto-provisioning stacji i serwisu (jesli nowa maszyna), detekcja dostepnych kodekov audio, inicjalizacja driverow audio na skonfigurowanych kartach, i uruchomienie serwera TCP. Ta feature pokrywa caly lifecycle daemona od startu do graceful shutdown.

## Aktorzy

| Aktor | Rola w tej feature |
|-------|-------------------|
| System (OS) | Uruchamia daemon (main) |
| Administrator | Konfiguruje rd.conf i baze danych |
| Unix signals | SIGHUP/SIGINT/SIGTERM do shutdown |

## Granica funkcjonalnosci

```
IN SCOPE:
  - Parsowanie argumentow CLI (-d debug flag)
  - Wczytanie konfiguracji (RDConfig / rd.conf)
  - Polaczenie z baza danych
  - Auto-provisioning stacji (STATIONS)
  - Auto-provisioning serwisu (SERVICES)
  - Detekcja kodekov (ProbeCaps): Ogg, FLAC, LAME, TwoLAME, MAD, MP4
  - Dynamiczne ladowanie bibliotek (TwoLAME, MAD via dlopen)
  - Czyszczenie wpisow driverow (ClearDriverEntries)
  - Inicjalizacja driverow audio (hpiInit, jackInit, alsaInit)
  - Inicjalizacja mixerow (InitMixers) → delegowane do CAE-004
  - Uruchomienie timera meteringu
  - Graceful shutdown (signal handling → exit w updateMeters)

OUT OF SCOPE:
  - TCP server i protokol → patrz CAE-001
  - Audio operations → patrz CAE-002, CAE-003, CAE-004
  - Metering logic → patrz CAE-005
```

## Reguly biznesowe (Gherkin)

```gherkin
Rule: Auto-provisioning stacji

  Scenario: Stacja nie istnieje w DB
    Given provisioningCreateHost == true
    And   template nazwy nie jest pusty
    And   stacja nie znaleziona w STATIONS
    When  daemon startuje
    Then  stacja tworzona z template (RDStation::create)
    And   short name ustawiane jesli skonfigurowane
    And   log INFO "created new host entry"

  Scenario: Provisioning wylaczony
    Given provisioningCreateHost == false
    When  daemon startuje
    Then  provisioning pomijany

  Scenario: Blad provisioning
    Given template podany ale tworzenie sie nie udaje
    When  daemon startuje
    Then  stderr "unable to provision host"
    And   exit(256)
  # Zrodlo: cae.cpp:1608-1671 | Pewnosc: potwierdzone

Rule: Detekcja kodekov

  Scenario: Kodek dostepny
    Given biblioteka kodeka zainstalowana w systemie
    When  ProbeCaps()
    Then  RDStation::setHaveCapability(true)

  Scenario: TwoLAME/MAD dynamicznie ladowane
    Given dlopen() moze zaladowac biblioteke
    When  LoadTwoLame() / LoadMad()
    Then  function pointers zaladowane
    And   capability = true

  Scenario: Kodek niedostepny
    Given biblioteka nie zainstalowana
    When  ProbeCaps()
    Then  capability = false (brak bledu)
  # Zrodlo: cae.cpp:1851-1905 | Pewnosc: potwierdzone

Rule: Graceful shutdown

  Scenario: Daemon otrzymuje SIGTERM
    Given daemon dziala
    When  SIGTERM/SIGINT/SIGHUP
    Then  global exiting = true
    And   nastepny cykl updateMeters():
    And   jackFree() + alsaFree() + hpiFree()
    And   log INFO "cae exiting"
    And   exit(0)
  # Zrodlo: cae.cpp:1515-1521, cae.h:101 | Pewnosc: potwierdzone

Rule: Startup failure

  Scenario: Port TCP zajety
    Given CAED_TCP_PORT juz uzyty
    When  cae_server->listen() zwraca false
    Then  log ERR "failed to bind port"
    And   exit(1)
  # Zrodlo: cae.cpp:154-158 | Pewnosc: potwierdzone
```

## API klas w scope

### MainObject (lifecycle)

**Konstruktor:**
| Metoda | Efekt |
|--------|-------|
| MainObject(parent, name) | Caly init: config, server, DB, provisioning, drivers, mixers, timer |

**Metody prywatne:**
| Metoda | Efekt |
|--------|-------|
| InitProvisioning() | Auto-tworzy stacje/serwis w DB |
| ProbeCaps(station) | Wykrywa codeki, zapisuje capabilities do DB |
| ClearDriverEntries(station) | Czysci wpisy kart/portow w DB |
| InitMixers() | Inicjalizuje hardware mixer z DB (→ CAE-004) |
| CheckLame() | Sprawdza dostepnosc LAME |
| CheckMp4Decode() | Sprawdza dostepnosc MP4 decodera |
| LoadTwoLame() | dlopen TwoLAME, laduje function pointers |
| LoadMad() | dlopen MAD, laduje function pointers |

**Driver Init/Free:**
| Metoda | Efekt |
|--------|-------|
| hpiInit(station) / hpiFree() | Inicjalizacja/zwolnienie HPI |
| jackInit(station) / jackFree() | Inicjalizacja/zwolnienie JACK |
| alsaInit(station) / alsaFree() | Inicjalizacja/zwolnienie ALSA |

**Globals:**
| Symbol | Typ | Opis |
|--------|-----|------|
| rd_config | RDConfig* | Konfiguracja (rd.conf) |
| SigHandler(int) | function | Unix signal handler |

## Acceptance Criteria (E2E)

```gherkin
Feature: Daemon Lifecycle

  Scenario: Normalny startup
    Given rd.conf z poprawna konfiguracja
    And   baza danych dostepna
    When  uruchomienie caed
    Then  konfiguracja wczytana
    And   DB polaczona
    And   provisioning wykonany (jesli wlaczony)

    And   drivery zainicjalizowane
    And   serwer TCP nasluchuje na CAED_TCP_PORT
    And   timer meteringu uruchomiony

  Scenario: Graceful shutdown
    Given daemon dziala
    When  kill -SIGTERM <pid>
    Then  drivery zwolnione
    And   daemon konczy z kodem 0

  Scenario: Auto-provisioning nowej stacji
    Given provisioningCreateHost == true
    And   stacja NIE istnieje w STATIONS
    When  daemon startuje
    Then  nowy wiersz w STATIONS z danymi z template
    And   log "created new host entry"
```
