# CAE-004: Audio Mixer & Routing Control

## Kontekst biznesowy

Stacja radiowa wymaga precyzyjnej kontroli glosnosci na wielu portach i streamach jednoczesnie. Ta feature obejmuje ustawianie poziomow wejscia/wyjscia, fade (plynna zmiana glosnosci w czasie), tryby kanalow (normal/swap/left/right), typ wejscia (analog/digital), passthrough miedzy portami oraz zrodlo zegara audio. Sa to operacje na hardware mixer, nie na strumieniu audio.

## Aktorzy

| Aktor | Rola w tej feature |
|-------|-------------------|
| Klient TCP | Steruje mixerem (volume, mode, passthrough, clock) |
| System (startup) | Inicjalizuje mixery z konfiguracji DB (InitMixers) |

## Granica funkcjonalnosci

```
IN SCOPE:
  - Input/output volume per stream (IV, OV)
  - Fade output volume w czasie (FV)
  - Input/output hardware levels per port (IL, OL)
  - Input/output mode (IM, OM) - Normal/Swap/Left/Right
  - Input VOX level (IX)
  - Input type (IT) - Analog/Digital
  - Input status query (IS)
  - Audio passthrough level (AL) - input→output routing
  - Clock source (CS)
  - Output status flag (OS)
  - Inicjalizacja mixerow przy starcie (InitMixers)

OUT OF SCOPE:
  - Playback/recording → patrz CAE-002, CAE-003
  - Metering → patrz CAE-005
```

  Scenario: Plynne fade

## Platform Independence

| Funkcja | Oryginal | Klon | Priorytet |
|---------|----------|------|-----------|
| Hardware volume | ALSA mixer / HPI mixer / JACK software | Software volume (0.0-1.0) | HIGH |
| Hardware levels | ALSA/HPI hardware levels (dB) | Software gain | HIGH |
| Audio passthrough | Hardware routing (port→port) | Software mixing | HIGH |
| Clock source | Hardware clock sync | N/A (software clock) | LOW |
| Input type | Hardware analog/digital switch | N/A | LOW |
