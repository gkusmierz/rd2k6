# LIB-013: Configuration & Bootstrap

## Kontekst biznesowy

Warstwa konfiguracji i bootstrap obejmuje inicjalizacje srodowiska Rivendell (RDApplication), parsowanie konfiguracji (RDConfig z /etc/rd.conf), polaczenie z baza danych, obsluge stacji roboczej i parametrow systemowych. Jest to pierwszy kod uruchamiany w kazdej aplikacji.

## Aktorzy

| Aktor | Rola |
|-------|------|
| System | Inicjalizacja srodowiska przy starcie |
| Administrator | Konfiguracja w /etc/rd.conf |

## Granica funkcjonalnosci

```
IN SCOPE:
  - RDApplication: singleton bootstrap (open, syslog, addTempFile)
  - RDConfig: parsowanie rd.conf
  - RDStation: konfiguracja stacji roboczej
  - RDSystem: parametry globalne
  - RDSqlQuery/RDOpenDb: polaczenie z baza danych
  - RDDbHeartbeat: monitoring DB
  - RDProfile: parser INI
  - RDCmdSwitch: parsowanie argumentow CLI
  - RDInstanceLock: blokada instancji
  - RDTempDirectory: katalogi tymczasowe
  - Konfiguracje modulow: RDAirPlayConf, RDLibraryConf, RDLogeditConf,
    RDCatchConf, RDMonitorConfig

OUT OF SCOPE:
  - Logika biznesowa modulow (inne FEAT)
```

## Reguly biznesowe (Gherkin)

```gherkin
Rule: Aplikacja waliduje srodowisko przy starcie
  Scenario: Brak polaczenia z DB
    Given brak serwera MySQL
    When RDApplication::open()
    Then ExitCode=ExitNoDb

  Scenario: Niezgodna wersja DB
    Given wersja schematu DB rozna od oczekiwanej
    When RDApplication::open()
    Then ErrorType=ErrorDbVersionSkew

  Scenario: Brak wpisu stacji
    Given stacja nie istnieje w tabeli STATIONS
    When RDApplication::open()
    Then ErrorType=ErrorNoHostEntry

Rule: Tylko jedna instancja procesu
  Scenario: Druga instancja
    Given rdairplay juz dziala
    When uruchomiony drugi rdairplay
    Then ExitCode=ExitPriorInstance
```
