# LIB-009: Podcast & RSS Feed Management

## Kontekst biznesowy

Zarzadzanie feedami podcastowymi (RSS) umozliwia publikowanie epizodow audio na zewnetrznych serwerach. Feed moze byc superfeedem agregujacym subfeed'y. System obsluguje upload audio, generowanie XML RSS, zarzadzanie obrazami, rendering logow do epizodow.

## Aktorzy

| Aktor | Rola |
|-------|------|
| Operator | Publikuje epizody, zarzadza feedami |
| System | Automatyczne publikowanie (autopost) |

## Granica funkcjonalnosci

```
IN SCOPE:
  - RDFeed: CRUD feed, publikacja, RSS XML generation
  - RDPodcast: CRUD epizod, upload/delete audio
  - RDRssSchemas: schematy RSS (Apple, Spotify, etc.)
  - Rendering logu do epizodu
  - Superfeed aggregation

OUT OF SCOPE:
  - Transfer plikow (LIB-011)
  - Konwersja audio (LIB-004)
```

## Reguly biznesowe (Gherkin)

```gherkin
Rule: Publikacja epizodu
  Scenario: Post cut jako epizod
    Given feed z serwerem SFTP
    And cut z audio
    When postCut()
    Then audio konwertowane, uploadowane, RSS updated

Rule: Superfeed
  Scenario: RSS agregacja
    Given superfeed z 3 subfeedami
    When rssXml()
    Then XML zawiera epizody ze wszystkich subfeedow

Rule: Shelf life
  Scenario: Epizod wygasa
    Given epizod z maxShelfLife=30 dni
    When minely 31 dni
    Then status = Expired
```
