## Opis

Wrapper wokol Linux PAM (Pluggable Authentication Modules). Umozliwia uwierzytelnianie uzytkownika systemowego przez nazwe i haslo (token).

## Klasa: RDPam

- **Nie dziedziczy z QObject** — plain C++ class
- **Plik naglowkowy**: lib/rdpam.h
- **Plik implementacji**: lib/rdpam.cpp
- **Zaleznosci**: security/pam_appl.h, RDApplication (rda->syslog)

### Konstruktor

| Sygnatura | Opis |
|-----------|------|
| `RDPam(const QString &pam_service)` | Tworzy obiekt PAM dla podanego serwisu (np. "login") |

### Metody publiczne

| Metoda | Zwraca | Opis |
|--------|--------|------|
| `authenticate(const QString &username, const QString &token)` | bool | Uwierzytelnia uzytkownika. Token = haslo. Zwraca true jesli sukces. |

### Metody prywatne

| Metoda | Zwraca | Opis |
|--------|--------|------|
| `CleanupPam()` | void | Zwalnia pamiec zaalokowana na odpowiedzi PAM |

### Pola prywatne

| Pole | Typ | Opis |
|------|-----|------|
| system_pam_service | QString | Nazwa serwisu PAM (np. "login") |
| system_token | QString | Haslo/token (przechowywane tymczasowo podczas uwierzytelniania) |
| system_pam_response | struct pam_response* | Bufor odpowiedzi PAM |
| system_pam_response_count | int | Liczba odpowiedzi |

### Funkcja globalna (friend)

| Funkcja | Opis |
|---------|------|
| `RDPamCallback(int num_msg, const struct pam_message **msg, struct pam_response **resp, void *appdata_ptr)` | Callback wymagany przez PAM API. Dostarcza haslo (token) dla PAM_PROMPT_ECHO_OFF. Loguje PAM_PROMPT_ECHO_ON i PAM_TEXT_INFO przez syslog. |

## Zachowanie kluczowe

- **authenticate()** flow:
  1. Ustawia system_token (haslo)
  2. Inicjalizuje pam_conv z callbackiem RDPamCallback
  3. pam_start() — otwiera sesje PAM
  4. pam_authenticate() — weryfikuje credentials
  5. pam_end() + CleanupPam() — sprzatanie
  6. Zwraca true/false
- **RDPamCallback**: przekazuje haslo (system_token) dla PAM_PROMPT_ECHO_OFF (typowe pytanie o haslo)
- **Logowanie**: bledy PAM logowane przez rda->syslog (LOG_WARNING)
- **Uzycie**: wywolywane przez RDSystemUser::validatePassword() z serwisem "login"
- Token jest przechowywany w obiekcie tylko na czas wywolania authenticate() (brak persystencji)
