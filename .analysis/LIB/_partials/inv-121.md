## Opis

Wrapper wokol QProcess, ktory dodaje identyfikator numeryczny (id) oraz emituje sygnaly started/finished z tym id. Umozliwia przechowywanie prywatnych danych (void*) i zbieranie stderr.

## Klasa: RDProcess

- **Dziedziczy**: QObject
- **Plik naglowkowy**: lib/rdprocess.h
- **Plik implementacji**: lib/rdprocess.cpp

### Konstruktor

| Sygnatura | Opis |
|-----------|------|
| `RDProcess(int id, QObject *parent=0)` | Tworzy kontener procesu z podanym identyfikatorem. Wewnetrznie tworzy QProcess i podlacza sygnaly. |

### Sygnaly

| Sygnal | Opis |
|--------|------|
| `started(int id)` | Emitowany gdy proces sie uruchomi (przekazuje id procesu) |
| `finished(int id)` | Emitowany gdy proces sie zakonczy (przekazuje id procesu) |

### Sloty (prywatne)

| Slot | Opis |
|------|------|
| `startedData()` | Reaguje na QProcess::started, emituje started(id) |
| `finishedData(int exit_code, QProcess::ExitStatus status)` | Reaguje na QProcess::finished, ustawia errorText (crash / exit code), emituje finished(id) |
| `readyReadStandardErrorData()` | Akumuluje dane z stderr procesu |

### Metody publiczne

| Metoda | Zwraca | Opis |
|--------|--------|------|
| `process()` | `QProcess*` | Dostep do wewnetrznego QProcess |
| `program()` | `QString` | Nazwa programu ustawiona przez start() |
| `arguments()` | `QStringList` | Argumenty programu ustawione przez start() |
| `prettyCommandString()` | `QString` | Czytelna reprezentacja komendy (program + argumenty) |
| `setProcessEnvironment(const QProcessEnvironment &env)` | void | Ustawia zmienne srodowiskowe procesu |
| `start(const QString &program, const QStringList &args)` | void | Uruchamia program z argumentami. Sprawdza istnienie pliku — jesli nie istnieje, ustawia errorText. |
| `errorText()` | `QString` | Tekst bledu — "ok" przy sukcesie, opis crash/exit code przy bledzie, "no such program" jesli plik nie istnieje |
| `privateData()` | `void*` | Pobiera prywatne dane uzytkownika |
| `setPrivateData(void *priv)` | void | Ustawia prywatne dane uzytkownika |
| `standardErrorData()` | `QByteArray` | Zbuforowane dane ze stderr procesu |

### Pola prywatne

| Pole | Typ | Opis |
|------|-----|------|
| p_id | int | Identyfikator procesu (przekazywany w sygnalach) |
| p_program | QString | Nazwa programu |
| p_arguments | QStringList | Argumenty |
| p_process | QProcess* | Wewnetrzny QProcess |
| p_error_text | QString | Tekst bledu |
| p_private_data | void* | Prywatne dane uzytkownika |
| p_standard_error_data | QByteArray | Bufor stderr |

## Zachowanie kluczowe

- Przy start() waliduje istnienie pliku programu przed uruchomieniem QProcess
- Po zakonczeniu procesu errorText zawiera "ok" (sukces), "process crashed" (crash) lub "process returned exit code N [stderr]" (bledny exit code)
- stderr jest akumulowane w buforze (nie kasowane miedzy odczytami)
- Destruktor usuwa QProcess
