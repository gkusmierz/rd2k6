## Opis

Rozszerzony QSqlQuery z automatycznym reconnectem i logowaniem bledow. Plus funkcja globalna RDOpenDb() do inicjalizacji polaczenia z baza MySQL.

## Klasa: RDSqlQuery

- **Dziedziczy**: QSqlQuery
- **Plik naglowkowy**: lib/rddb.h
- **Plik implementacji**: lib/rddb.cpp
- **Nie jest QObject** — brak sygnalow/slotow

### Konstruktor

| Sygnatura | Opis |
|-----------|------|
| `RDSqlQuery(const QString &query = QString::null, bool reconnect=true)` | Wykonuje zapytanie SQL. Jesli query nie jest aktywne i reconnect=true, probuje ponownie otworzyc polaczenie i powtorzyc zapytanie. Parsuje liczbe kolumn dla SELECT. |

### Metody publiczne

| Metoda | Zwraca | Opis |
|--------|--------|------|
| `columns()` | int | Liczba kolumn w wyniku SELECT (parsowana z tekstu zapytania) |
| `value(int index)` | QVariant | Rozszerzone value() — loguje na stderr jesli wynik nie jest valid |

### Metody statyczne

| Metoda | Zwraca | Opis |
|--------|--------|------|
| `run(const QString &sql, bool *ok=NULL)` | QVariant | Wykonuje SQL i zwraca lastInsertId. ok wskazuje czy zapytanie bylo aktywne. |
| `apply(const QString &sql, QString *err_msg=NULL)` | bool | Wykonuje SQL, zwraca true jesli aktywne. Przy bledzie zapisuje komunikat do err_msg. |
| `rows(const QString &sql)` | int | Wykonuje SQL i zwraca size() wyniku (liczba wierszy) |

### Pola prywatne

| Pole | Typ | Opis |
|------|-----|------|
| sql_columns | int | Sparsowana liczba kolumn z zapytania SELECT |

## Zachowanie kluczowe

- **Auto-reconnect**: jesli pierwsze wykonanie query nie powiedzie sie, probuje db.open() i powtarza query
- **Logowanie bledow**: bledy SQL logowane na stderr + syslog (przez rda->syslog)
- **Parsowanie kolumn**: dla zapytania SELECT zlicza nazwy kolumn miedzy SELECT a FROM (split po spacjach i przecinkach)
- **value()**: dodatkowa walidacja — loguje na stderr jesli wartosc nie jest valid

## Funkcja globalna: RDOpenDb

| Sygnatura | Opis |
|-----------|------|
| `bool RDOpenDb(int *schema, QString *err_str, RDConfig *config)` | Otwiera polaczenie MySQL, tworzy heartbeat, zwraca numer schematu DB |

### Zachowanie RDOpenDb

1. Jesli polaczenie nie jest otwarte — tworzy nowe z parametrow RDConfig (driver, hostname, dbname, username, password)
2. Tworzy `RDDbHeartbeat` z config->mysqlHeartbeatInterval()
3. Ustawia kodowanie: `SET NAMES utf8mb4 COLLATE utf8mb4_general_ci`
4. Pobiera numer schematu z tabeli VERSION (kolumna DB)
   - Jesli tabela VERSION istnieje: schema = wartosc DB
   - Jesli tabela VERSION nie istnieje ale inne tabele sa: schema = -1
   - Jesli baza jest pusta: schema = 0
5. Zwraca true przy sukcesie, false z komunikatem bledu przy porazce
