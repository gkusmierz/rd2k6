# RDCutDialog

## Purpose
Modal dialog for browsing and selecting a specific cut (audio segment) within a cart. Shows a two-panel view: carts on top, cuts for selected cart below. Supports filtering, optional "clear" selection, adding new carts, and voice track exclusion.

## Inheritance
- **Parent class:** RDDialog (QDialog)
- **Qt meta:** Q_OBJECT

## Constructor
```cpp
RDCutDialog(QString *cutname, const QString &caption, QString *filter = 0,
            QString *group = 0, QString *schedcode = NULL, bool show_clear = false,
            bool allow_add = false, bool exclude_tracks = false, QWidget *parent = 0);
```
- `cutname` — output: selected cut name (cart_number + cut_number format)
- `caption` — dialog window title
- `filter` — persistent text filter (in/out)
- `group` — persistent group filter (in/out)
- `schedcode` — persistent scheduler code filter (in/out)
- `show_clear` — whether to show a "Clear" button allowing empty selection
- `allow_add` — whether to show an "Add" button for creating new carts
- `exclude_tracks` — whether to exclude voice track carts from the list

## Public API / Slots

| Method | Signature | Description |
|--------|-----------|-------------|
| `exec` | `int exec()` | Opens the dialog; returns QDialog result code. Selected cut name written to constructor's `cutname` pointer. |
| `sizeHint` | `QSize sizeHint() const` | Preferred dialog size |
| `sizePolicy` | `QSizePolicy sizePolicy() const` | Size policy |

## Signals
None.

## Private Slots (UI Interactions)
- `clearData()` — clear selection (empty cut)
- `groupActivatedData(const QString &)` — group combo changed
- `limitChangedData(int state)` — results limit checkbox toggled
- `cartClickedData(Q3ListViewItem *)` — cart selected, refreshes cut list
- `selectionChangedData()` — cut selection changed
- `searchButtonData()` — search button clicked
- `clearButtonData()` — clear filter button clicked
- `addButtonData()` — add new cart button clicked
- `okData()` / `cancelData()` — OK/Cancel buttons

## Key Widgets
- `cut_cart_list` (RDListView) — cart list view (top panel)
- `cut_cut_list` (RDListView) — cut list view (bottom panel, shows cuts of selected cart)
- `cut_filter_edit` (QLineEdit) — text filter
- `cut_group_box` (QComboBox) — group filter dropdown
- `cut_schedcode_box` (QComboBox) — scheduler code filter dropdown
- `cart_limit_box` (QCheckBox) — results limit toggle

## State Persistence
Dialog saves/restores filter state via `LoadState()`/`SaveState()`.

## Dependencies
- RDCart, RDListView, RDDialog
- Uses separate playout/macro column maps for the list views
