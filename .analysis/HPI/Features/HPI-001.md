# HPI-001: Sound Card Management & Mixer Control

## Kontekst biznesowy

Zarzadzanie kartami dzwiekowymi AudioScience HPI stanowi fundament calej biblioteki. Ta feature odpowiada za wykrywanie zainstalowanych kart, rejestracje mixer controls, kontrole glosnosci/poziomu/routing, monitorowanie AES/EBU errors i udostepnianie informacji o kartach (serial, wersje firmware). Bez tej feature, playback i recording nie moga dzialac.

## Aktorzy

| Aktor | Rola w tej feature |
|-------|-------------------|
| CAE (daemon) | Tworzy instancje SoundCard, ustawia volume/level, monitoruje metery |
| RPC (daemon) | Tworzy instancje SoundCard do kontroli sprzetowej |
| System Administrator | Konfiguruje clock source, input mux |

## Granica funkcjonalnosci

```
IN SCOPE:
  - Enumeracja kart HPI (HPIProbe)
  - Kontrola mixer: volume, level, mode, mux, passthrough, VOX
  - Fading z profilem (Linear/Log)
  - Metering (peak meters na strumieniach i portach)
  - AES/EBU error monitoring
  - Informacje o kartach (serial, wersje)
  - Widget UI do wyboru karty/portu (RDHPISoundSelector)
OUT OF SCOPE:
  - Streaming audio (playback/recording) -> patrz HPI-002, HPI-003
  - Tuner API (zadeklarowane ale nie zaimplementowane)
```

## Reguly biznesowe (Gherkin)

```gherkin
Rule: Bounds checking na indeksach sprzetowych
  Scenario: Zapytanie o nieistniejacy adapter
    Given indeks karty card >= HPI_MAX_ADAPTERS
    When haveInputVolume(card, stream, port) jest wywolane
    Then zwraca false
  # Zrodlo: rdhpisoundcard.cpp:206-211 | Pewnosc: potwierdzone

Rule: Volume symetryczny na oba kanaly
  Scenario: Ustawienie glosnosci wyjscia
    Given karta card, strumien stream, port port, level
    When setOutputVolume(card, stream, port, level) jest wywolane
    Then gain[0] = level (lewy kanal)
    And gain[1] = level (prawy kanal)
    And HPI_VolumeSetGain jest wywolane z gain[]
    But jesli haveOutputVolume(card, stream, port) = false, operacja ignorowana
  # Zrodlo: rdhpisoundcard.cpp:559-570 | Pewnosc: potwierdzone

Rule: Fade profil
  Scenario: Fade output z profilem logarytmicznym
    Given fade_type = Log (domyslna wartosc)
    When fadeOutputVolume(card, stream, port, level, length) jest wywolane
    Then HPI_VolumeAutoFadeProfile z HPI_VOLUME_AUTOFADE_LOG
    And fade trwa length ms
  # Zrodlo: rdhpisoundcard.cpp:574-587 | Pewnosc: potwierdzone

Rule: AES/EBU error monitoring
  Scenario: Zmiana stanu bledu na porcie AES/EBU
    Given clock timer taktuje co 20ms
    When error_word na porcie zmieni sie vs poprzednia wartosc
    Then inputPortError(card, port) jest emitowany
  # Zrodlo: rdhpisoundcard.cpp:671-686 | Pewnosc: potwierdzone

Rule: Multiplekser portu wejsciowego
  Scenario: Zmiana zrodla na multiplekserze
    Given port wejsciowy z mux (haveInputPortMux = true)
    When setInputPortMux(card, port, node) jest wywolane
    Then akceptuje tylko LineIn lub AesEbuIn jako node
    And dla innych node zwraca false
  # Zrodlo: rdhpisoundcard.cpp:266-290 | Pewnosc: potwierdzone
```
