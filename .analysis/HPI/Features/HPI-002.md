# HPI-002: Audio Playback via HPI

## Kontekst biznesowy

Odtwarzanie plikow audio przez karty AudioScience HPI jest jednym z dwoch glownych use cases biblioteki. Ta feature zapewnia streaming danych audio z plikow WAV/MPEG do HPI output stream z pelna kontrola transportu (play/pause/stop), pozycjonowaniem, kontrola predkosci (timescaling) i automatycznym zatrzymaniem po zakonczeniu pliku. Jest uzywana przez CAE (Core Audio Engine) do profesjonalnego odtwarzania w radiu.

## Aktorzy

| Aktor | Rola w tej feature |
|-------|-------------------|
| CAE (daemon) | Tworzy PlayStream, kontroluje transport (play/pause/stop), monitoruje pozycje |

## Granica funkcjonalnosci

```
IN SCOPE:
  - Otwarcie/zamkniecie pliku audio (WAV, MPEG)
  - Kontrola transportu: play, pause, stop
  - Pozycjonowanie (seek) podczas odtwarzania i pauzy
  - Kontrola predkosci (timescaling, pitch variation)
  - Raportowanie pozycji (position signal)
  - Auto-stop po wyczerpaniu danych (DRAINED)
  - Ograniczenie czasu odtwarzania (play_length)
  - Wsparcie formatow: PCM 8/16/24/32, MPEG L1/L2/L3, Vorbis
  - Mutex na strumieniach output (wspoldzielenie miedzy instancjami)
OUT OF SCOPE:
  - Mixer control (volume, level) -> patrz HPI-001
  - Nagrywanie -> patrz HPI-003
```

## API klas w scope

### RDHPIPlayStream

**Odpowiedzialnosc:** Odtwarzanie plikow audio (WAV/MPEG) przez HPI output stream. Dziedziczy z QObject i RDWaveFile (multiple inheritance).

**Publiczne API:**
| Metoda | Parametry | Efekt | Warunki |
|--------|-----------|-------|---------|
| openWave | -- | Otwiera WAV z ustawionej nazwy, alokuje HPI stream | !is_open |
| openWave | QString filename | Ustawia nazwe i otwiera | !is_open |
| closeWave | -- | Zamyka plik, zwalnia stream (stop jesli gra) | -- |
| formatSupported | RDWaveFile::Format | Sprawdza wsparcie formatu na karcie | card_number >= 0 |
| formatSupported | -- | Sprawdza wsparcie aktualnego formatu | -- |
| getCard | -- | Zwraca numer karty | -- |
| getStream | -- | Zwraca numer strumienia | -- |
| getSpeed | -- | Zwraca predkosc (1000 = 100%) | -- |
| setSpeed | int speed, bool pitch, bool rate | Ustawia predkosc | Ograniczenia zakresu |
| getState | -- | Zwraca State (Stopped/Playing/Paused) | -- |
| errorString | Error err | Opis bledu tekstowo | -- |

**Sloty (public):**
| Slot | Parametry | Efekt |
|------|-----------|-------|
| setCard | int card | Ustawia karte (tylko gdy !playing) |
| play | -- | Rozpoczyna odtwarzanie, zwraca bool |
| pause | -- | Wstrzymuje odtwarzanie |
| stop | -- | Zatrzymuje, resetuje pozycje |
| currentPosition | -- | Zwraca aktualna pozycje (samples) |
| setPosition | unsigned samples | Seek (z restart transport jesli gra) |
| setPlayLength | int length | Max czas odtwarzania (ms, 0=bez limitu) |
| tickClock | -- | Timer callback 50ms -- streamuje dane |

**Sygnaly:**
| Sygnal | Parametry | Znaczenie |
|--------|-----------|-----------|
| isStopped | bool state | true=zatrzymany, false=gra |
| played | -- | Odtwarzanie rozpoczete |
| paused | -- | Odtwarzanie wstrzymane |
| stopped | -- | Odtwarzanie zatrzymane |
| position | int samples | Aktualna pozycja w samplach |
| stateChanged | int card, int stream, int state | Zmiana stanu (0/1/2) |

**Enums:**
| Enum | Wartosci | Znaczenie |
|------|----------|-----------|
| State | Stopped=0, Playing=1, Paused=2 | Stan odtwarzania |
| Error | Ok=0, NoFile=1, NoStream=2, AlreadyOpen=3 | Kody bledow |

## Acceptance Criteria (E2E)

```gherkin
Feature: Audio Playback via HPI
  Scenario: Pelne odtworzenie pliku WAV
    Given karta HPI jest dostepna
    And plik "test.wav" istnieje (PCM16, 44100Hz, stereo)
    When setCard(0) i openWave("test.wav") i play()
    Then state = Playing
    And position() emitowany co ~150ms
    And po zakonczeniu pliku state = Stopped automatycznie (DRAINED)

  Scenario: Pause i resume
    Given strumien odtwarza na pozycji 50000 sampli
    When pause() jest wywolane
    Then state = Paused, paused() emitowany
    When play() jest wywolane ponownie
    Then state = Playing, odtwarzanie kontynuuje od pozycji pauzy
```
