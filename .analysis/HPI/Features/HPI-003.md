# HPI-003: Audio Recording via HPI

## Kontekst biznesowy

Nagrywanie audio z kart AudioScience HPI jest drugim glownym use case biblioteki. Ta feature zapewnia pelny cykl nagrywania: arm (recordReady) -> record -> pause -> stop, z automatyczna detekcja poczatku nagrywania (RecordStarted), obsluga VOX (Voice Operated eXchange), limitem czasu nagrywania i zapisem do plikow WAV/MPEG. Jest uzywana przez CAE do nagrywania w studiu i automatycznego nagrywania audycji.

## Aktorzy

| Aktor | Rola w tej feature |
|-------|-------------------|
| CAE (daemon) | Tworzy RecordStream, kontroluje cykl nagrywania, monitoruje pozycje |

## Granica funkcjonalnosci

```
IN SCOPE:
  - Tworzenie pliku audio (WAV, MPEG)
  - Cykl nagrywania: arm (recordReady) -> record -> pause -> stop
  - Detekcja poczatku nagrywania (RecordStarted signal)
  - VOX (Voice Operated eXchange) -- delegowane do SoundCard
  - Ograniczenie czasu nagrywania (record_length)
  - Raportowanie pozycji (position signal)
  - Wsparcie formatow: PCM 8/16/24/32, MPEG L1/L2/L3, Vorbis (jako PCM16)
  - MEXT chunk konfiguracja dla MPEG
OUT OF SCOPE:
  - Mixer control -> patrz HPI-001
  - Odtwarzanie -> patrz HPI-002
```

## Reguly biznesowe (Gherkin)

```gherkin
Rule: Maszyna stanow nagrywania (5 stanow)
  Scenario: Pelny cykl nagrywania
    Given strumien jest Stopped
    When recordReady() jest wywolane
    Then stan = RecordReady, ready() emitowany
    When record() jest wywolane
    Then stan = Recording, recording() emitowany
    When samples_recorded > 0 (pierwsze dane)
    Then stan = RecordStarted, recordStart() emitowany (jednorazowo)
    When pause() jest wywolane
    Then stan = Paused, HPI stream restartowany (armed)
    When stop() jest wywolane
    Then stan = Stopped, dane zapisane do pliku
  # Zrodlo: rdhpirecordstream.cpp:354-369, 387-644 | Pewnosc: potwierdzone

Rule: Zmiana karty tylko gdy nie nagrywa
  Scenario: Zmiana karty podczas nagrywania
    Given is_recording = true
    When setCard(card) jest wywolane
    Then karta NIE jest zmieniana
  # Zrodlo: rdhpirecordstream.cpp:325-333 | Pewnosc: potwierdzone

Rule: Podwojne otwarcie blokowane
  Scenario: Tworzenie pliku gdy juz otwarty
    Given is_open = true
    When createWave() jest wywolane
    Then zwraca AlreadyOpen
  # Zrodlo: rdhpirecordstream.cpp:144-148 | Pewnosc: potwierdzone

Rule: Auto-record gdy nie uzbrojony
  Scenario: Record bez recordReady
    Given strumien nie jest w stanie ready
    When record() jest wywolane
    Then recordReady() jest wywolane automatycznie przed record
  # Zrodlo: rdhpirecordstream.cpp:572-574 | Pewnosc: potwierdzone

Rule: RecordStarted jednorazowe
  Scenario: Detekcja poczatku nagrywania
    Given strumien nagrywa (is_recording = true)
    And record_started = false
    When tickClock() sprawdza samples_recorded > 0
    Then recordStart() emitowany
    And stateChanged(RecordStarted) emitowany
    And record_started = true (nie emitowane ponownie)
  # Zrodlo: rdhpirecordstream.cpp:664-676 | Pewnosc: potwierdzone

Rule: Pause restartuje HPI stream
  Scenario: Pauza nagrywania
    Given strumien nagrywa
    When pause() jest wywolane
    Then HPI InStream zatrzymany i restartowany
    And is_recording = false, is_paused = true
    And HPI stream dalej zbiera dane (ale NIE zapisywane do pliku)
  # Zrodlo: rdhpirecordstream.cpp:593-614 | Pewnosc: potwierdzone

Rule: Fragment size ograniczenie (ALSA compatibility)
  Scenario: Obliczenie rozmiaru fragmentu
    Given bufor HPI ma rozmiar buffer_size
    When fragment_size jest obliczany
    Then fragment_size = min(buffer_size/4, 192000)
  # Zrodlo: rdhpirecordstream.cpp:409-411 | Pewnosc: potwierdzone

Rule: MEXT chunk dla MPEG
  Scenario: Nagrywanie w formacie MPEG
    Given format = MPEG Layer 2
    And getMextChunk() = true
    When recordReady() konfiguruje format
    Then MEXT chunk jest ustawiony: homogenous=true, paddingUsed=false
    And frameSize = 144 * bitrate / sampleRate
    And ancillaryLength = 5
  # Zrodlo: rdhpirecordstream.cpp:496-511 | Pewnosc: potwierdzone

Rule: Auto-pause po record_length
  Scenario: Ograniczony czas nagrywania
    Given record_length = 60000 (60 sekund)
    When samples zaczynaja plynac (RecordStarted)
    Then length_timer startuje na 60000ms
    And po 60 sekundach pause() jest wywolane automatycznie

  Scenario: Wlaczenie diagnostyki
```

## API klas w scope

### RDHPIRecordStream

**Odpowiedzialnosc:** Nagrywanie audio z kart HPI do plikow WAV/MPEG. Dziedziczy z QObject i RDWaveFile (multiple inheritance).

**Publiczne API:**
| Metoda | Parametry | Efekt | Warunki |
|--------|-----------|-------|---------|
| createWave | -- | Tworzy WAV i alokuje HPI input stream | !is_open |
| createWave | QString filename | Ustawia nazwe i tworzy | !is_open |
| closeWave | -- | Stop jesli nagrywa, zamyka plik, zwalnia stream | -- |
| formatSupported | RDWaveFile::Format | Sprawdza wsparcie formatu na karcie | card_number >= 0 |
| formatSupported | -- | Sprawdza wsparcie aktualnego formatu | -- |
| getCard | -- | Zwraca numer karty | -- |
| getStream | -- | Zwraca numer strumienia | -- |
| haveInputVOX | -- | Sprawdza dostepnosc VOX (deleguje do SoundCard) | -- |
| getState | -- | Zwraca RecordState (5 stanow) | -- |
| getPosition | -- | Zwraca pozycje w samplach (0 jesli Stopped) | -- |
| samplesRecorded | -- | Zwraca liczbe nagranych sampli | -- |
| errorString | Error err | Opis bledu tekstowo | -- |

**Sloty (public):**
| Slot | Parametry | Efekt |
|------|-----------|-------|
| setCard | int card | Ustawia karte (tylko gdy !is_recording) |
| setStream | int stream | Ustawia numer strumienia |
| recordReady | -- | Arm nagrywania: konfiguruje format, startuje HPI |
| record | -- | Rozpoczyna nagrywanie (auto-arm jesli nie ready) |
| pause | -- | Wstrzymuje nagrywanie, restartuje HPI |
| stop | -- | Zatrzymuje nagrywanie, zwalnia bufor |
| setInputVOX | int gain | Ustawia prog VOX (deleguje do SoundCard) |
| setRecordLength | int length | Max czas nagrywania (ms, 0=bez limitu) |

**Sloty (private):**
| Slot | Efekt |
|------|-------|
| tickClock | Timer callback 100ms -- odczytuje dane z HPI i zapisuje do pliku |

**Sygnaly:**
| Sygnal | Parametry | Znaczenie |
|--------|-----------|-----------|
| isStopped | bool state | true=zatrzymany, false=nagrywa |
| ready | -- | Strumien uzbrojony (armed) |
| recording | -- | Nagrywanie rozpoczete |
| recordStart | -- | Pierwsze sample nagrane (actual data flowing) |
| paused | -- | Nagrywanie wstrzymane |
| stopped | -- | Nagrywanie zatrzymane |
| position | int samples | Aktualna pozycja w samplach |
| stateChanged | int card, int stream, int state | 0=Recording, 1=Ready, 2=Paused, 3=Stopped, 4=Started |

**Enums:**
| Enum | Wartosci | Znaczenie |
|------|----------|-----------|
| RecordState | Recording=0, RecordReady=1, Paused=2, Stopped=3, RecordStarted=4 | Stan nagrywania |
| Error | Ok=0, NoFile=1, NoStream=2, AlreadyOpen=3 | Kody bledow |

## Acceptance Criteria (E2E)

```gherkin
Feature: Audio Recording via HPI
  Scenario: Pelne nagranie audio do pliku WAV
    Given karta HPI jest dostepna na karcie 0, stream 0
    And plik wyjsciowy "output.wav" moze byc utworzony

    Given record_length = 3000 (3 sekundy)
    When record() jest wywolane
```
