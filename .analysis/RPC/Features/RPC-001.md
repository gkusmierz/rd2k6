# RPC-001: Daemon Core & IPC Server

## Kontekst biznesowy

ripcd jest centralnym demonem IPC systemu Rivendell -- kazda stacja robocza uruchamia jedna instancje. Daemon Core odpowiada za: inicjalizacje aplikacji (RDApplication), nasluchiwanie na porcie TCP (RIPC protocol), zarzadzanie polaczeniami klientow (autentykacja, dispatch komend, broadcast), oraz inicjalizacje infrastruktury (sokety UDP RML, timery, multicast notyfikacje). Bez tego fundamentu zadna inna feature ripcd nie moze dzialac.

## Aktorzy

| Aktor | Rola w tej feature |
|-------|-------------------|
| Aplikacje Rivendell | Lacza sie via TCP, wysylaja komendy, odbieraja broadcast |
| ripcd daemon | Singleton serwer, akceptuje polaczenia, dispatchuje komendy |

## Granica funkcjonalnosci

```
IN SCOPE:
  - MainObject singleton lifecycle (konstruktor, destruktor)
  - QTcpServer nasluchiwanie na RIPCD_TCP_PORT (5006)
  - RipcdConnection management (new, auth, close, garbage collection)
  - Protokol RIPC: DC, PW, RU, SU, MS (DispatchCommand)
  - BroadcastCommand i EchoCommand
  - Sokety UDP RML (bind echo/noecho/reply)
  - Multicast notifications (RDMulticaster)
  - Exit/signal handling (SIGTERM, SIGINT, SIGCHLD)
  - Debug mode (-d flag)
OUT OF SCOPE:
  - Przetwarzanie komend RML -> patrz RPC-003
  - GPIO state management -> patrz RPC-002
  - Switcher driver loading -> patrz RPC-004
  - TTY management -> patrz RPC-005
  - JACK integration -> patrz RPC-006
```

## API klas w scope

### MainObject

**Odpowiedzialnosc:** Singleton demona -- zarzadza lifecycle IPC, nasluchuje TCP/UDP, dispatchuje komendy.

**Kluczowe sloty:**

| Slot | Parametry | Efekt | Warunki |
|------|-----------|-------|---------|
| newConnectionData() | - | Akceptuje nowe polaczenie TCP, tworzy RipcdConnection, laczy ready/kill mapper | - |
| readyReadData(conn_id) | int | Czyta dane z soketu klienta, akumuluje w conn->accum, wywoluje DispatchCommand | conn musi istniec |
| killData(conn_id) | int | Zamyka polaczenie klienta, ustawia closing flag | - |
| garbageData() | - | Usuwa polaczenia z flagami closing | - |
| exitTimerData() | - | Sprawdza sygnaly SIGTERM/SIGINT, graceful shutdown | - |
| notificationReceivedData(msg, addr) | QString, QHostAddress | Broadcastuje notyfikacje multicast do klientow | - |

**Kluczowe metody prywatne:**

| Metoda | Parametry | Efekt |
|--------|-----------|-------|
| DispatchCommand(conn) | RipcdConnection* | Parsuje i wykonuje komende RIPC (DC/PW/RU/SU/MS) |
| EchoCommand(conn_id, cmd) | int, QString | Wysyla odpowiedz do jednego klienta |
| BroadcastCommand(cmd, except) | QString, int | Wysyla do wszystkich uwierzytelnionych klientow |
| SetUser(username) | QString | Zmienia uzytkownika, broadcastuje RU |

### RipcdConnection

**Odpowiedzialnosc:** Reprezentuje jedno polaczenie TCP klienta RIPC.

**Publiczne API:**

| Metoda | Parametry | Efekt | Warunki |
|--------|-----------|-------|---------|
| id() | - | Zwraca ID polaczenia | - |
| socket() | - | Zwraca QTcpSocket* | - |
| isAuthenticated() | - | Czy sesja uwierzytelniona | - |
| setAuthenticated(state) | bool | Ustaw stan autentykacji | - |
| isClosing() | - | Czy polaczenie zamykane | - |
| close() | - | Zamknij polaczenie (graceful) | - |

**Pola publiczne:**
- accum (QString) -- bufor akumulacyjny komend

## Protokoly komunikacji

### RIPC Protocol (TCP:5006)

**Unprivileged:**

| Komenda | Parametry | Odpowiedz | Znaczenie |
|---------|-----------|-----------|-----------|
| DC | - | (zamyka) | Drop Connection |
| PW | passwd | PW +! / PW -! | Autentykacja |

**Privileged:**

| Komenda | Parametry | Odpowiedz | Znaczenie |
|---------|-----------|-----------|-----------|
| RU | - | RU username! | Request User |
| SU | username | (broadcast RU username!) | Set User |
| MS | ip_addr echo rml | - | RML Send |

### UDP Sockets (RML)

| Port | Nazwa | Kierunek | Opis |
|------|-------|----------|------|
| 5858 | RD_RML_ECHO_PORT | IN | RML z echo |
| 5859 | RD_RML_NOECHO_PORT | IN | RML bez echo |
| 5860 | RD_RML_REPLY_PORT | IN/OUT | Odpowiedzi RML |
