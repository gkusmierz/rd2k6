# RPC-005: Serial/TTY Communication & Code Traps

## Kontekst biznesowy

Wiele urzadzen audio komunikuje sie ze stacja robocza przez porty szeregowe (RS-232/485). ripcd zarzadza portami TTY: otwiera je na starcie wedlug konfiguracji DB, obsluguje wysylanie danych (string i binary), odczytywanie z code traps (wzorce danych triggerujace makro cart) i restart portow. To jest infrastruktura komunikacji serialowej uzywana przez drivery switcher.

## Aktorzy

| Aktor | Rola w tej feature |
|-------|-------------------|
| Operator | Steruje portami serial via RML |
| Drivery serial | Uzywaja portow TTY do komunikacji ze sprzetem |

## Granica funkcjonalnosci

```
IN SCOPE:
  - LoadLocalMacros() -- ladowanie konfiguracji TTY z DB
  - ripcd_tty_dev[], ripcd_tty_inuse[], ripcd_tty_term[]
  - ripcd_tty_trap[] (RDCodeTrap)
  - ttyReadyReadData -- dane od portu serial
  - ttyTrapData -- code trap wyzwolony
  - RML komendy: BO (binary output), SO (serial output), SI (serial input trap),
    SC (serial clear trap), SY (sync/restart TTY)
OUT OF SCOPE:
  - Drivery uzywajace TTY -> patrz RPC-004
  - GPIO management -> patrz RPC-002
```

## Data Model (tabele DB w scope)

| Tabela | Kolumna | Typ | Opis |
|--------|---------|-----|------|
| TTYS | STATION_NAME | varchar | FK do stacji |
| TTYS | PORT_ID | int | ID portu (0..MAX_TTYS-1) |
| TTYS | PORT | varchar | Sciezka urzadzenia (/dev/ttyS0) |
| TTYS | BAUD_RATE | int | Predkosc transmisji |
| TTYS | DATA_BITS | int | Bity danych |
| TTYS | PARITY | int | Parzystosc (RDTTYDevice::Parity) |
| TTYS | TERMINATION | int | Typ zakonczenia (RDTty::Termination) |
| TTYS | ACTIVE | varchar | "Y" / "N" |
