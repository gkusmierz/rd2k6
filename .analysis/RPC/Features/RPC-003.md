# RPC-003: RML Command Processing & Routing

## Kontekst biznesowy

RML (Rivendell Macro Language) to jezyk komend automatyzacji radiowej. ripcd jest centralnym wykonawca komend RML -- odbiera je z trzech soketow UDP (echo, noecho, reply) oraz z klientow RIPC (komenda MS). Komendy moga byc wykonywane lokalnie (RunLocalMacros) lub forwardowane do zdalnych stacji via UDP. Komendy switcher/GPIO sa delegowane do odpowiednich driverow. Ta feature obejmuje infrastrukture przetwarzania RML, routing local/remote, oraz komendy niezwiazane z GPIO ani driverami.

## Aktorzy

| Aktor | Rola w tej feature |
|-------|-------------------|
| Operator | Wysyla komendy RML (via aplikacje) |
| Aplikacje Rivendell | Zrodlo komend RML via RIPC MS lub UDP |
| Zdalne stacje ripcd | Cel forwardingu RML |

## Granica funkcjonalnosci

```
IN SCOPE:
  - ReadRmlSocket -- czytanie komend z soketow UDP
  - sendRml -- wysylanie RML via UDP
  - RunLocalMacros -- dispatch switch(rml->command())
  - ForwardConvert -- backward compatibility
  - RML routing: local loopback vs remote forwarding
  - Komendy: LO (login), MB (popup), MT (macro timer), RN (run external), UO (UDP output)
  - Macro timers (ripc_macro_timer[], macroTimerData)
OUT OF SCOPE:
  - Komendy GPIO (GI, GE, TA) -> patrz RPC-002
  - Komendy serial (BO, SI, SC, SO, SY) -> patrz RPC-005
  - Komendy JACK (JC, JD, JZ) -> patrz RPC-006
  - Komendy switcher (CL, FS, GO, ST, SA, SD, SG, SR, SL, SX) -> delegowane do RPC-004
```

    And   BroadcastCommand informuje klientow

## API klas w scope

### MainObject (metody RML)

| Metoda | Parametry | Efekt |
|--------|-----------|-------|
| ReadRmlSocket(sock, role, echo) | QUdpSocket*, RDMacro::Role, bool | Czyta komende RML z UDP soketu |
| sendRml(rml) | RDMacro* | Wysyla RML via UDP do docelowego hosta |
| RunLocalMacros(rml) | RDMacro* | Wykonuje komende RML lokalnie (dispatch switch) |
| ForwardConvert(rml) | const RDMacro& | Konwertuje stare formaty RML do aktualnych |
| macroTimerData(num) | int | Timer wygasl -- uruchamia ExecCart |
| RunCommand(uid, gid, cmd) | uid_t, gid_t, QString | Uruchamia zewnetrzny proces z uid/gid |
