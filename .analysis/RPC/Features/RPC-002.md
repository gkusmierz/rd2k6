# RPC-002: GPIO Management & Macro Cart Mapping

## Kontekst biznesowy

GPIO (General Purpose Input/Output) to mechanizm automatyzacji radiowej -- piny fizyczne na sprzecie audio moga triggerowal akcje w systemie. ripcd zarzadza stanem GPIO dla wszystkich macierzy, mapuje piny na makro carty (automatyczne uruchamianie akcji przy zmianie stanu), obsluguje maskowanie pinow i broadcastuje zmiany do wszystkich polaczonych aplikacji. To jest "serce automatyzacji" stacji radiowej.

## Aktorzy

| Aktor | Rola w tej feature |
|-------|-------------------|
| Sprzet audio/GPIO | Zrodlo zdarzen GPIO (zmiana stanu pinow) |
| Operator | Konfiguruje mapowanie GPIO -> cart, maskowanie |
| Aplikacje Rivendell | Odbieraja broadcast zmian GPIO |

## Granica funkcjonalnosci

```
IN SCOPE:
  - Tablice stanu GPIO: ripcd_gpi_state[][], ripcd_gpo_state[][]
  - Tablice makr GPIO: ripcd_gpi_macro[][], ripcd_gpo_macro[][]
  - Tablice masek GPIO: ripcd_gpi_mask[][], ripcd_gpo_mask[][]
  - LoadGpiTable() -- ladowanie mapowania z DB
  - gpiChangedData / gpoChangedData -- reakcja na zmiane GPIO
  - gpiStateData / gpoStateData -- raport pelnego stanu
  - RML komendy: GI (set cart), GE (set mask)
  - Broadcast: GI, GO, GC, GD, GM, GN
  - LogGpioEvent -- audit trail
  - ExecCart -- uruchamianie makro cart
  - SendGpi/SendGpo/SendGpiMask/SendGpoMask/SendGpiCart/SendGpoCart
  - On-Air flag (TA command)
OUT OF SCOPE:
  - Fizyczna komunikacja z GPIO hardware -> patrz RPC-004 (drivery)
  - Inne komendy RML -> patrz RPC-003
```

```gherkin

    And   ripcd_gpi_macro[0][3][1] == 12345

    Then  ripcd_gpi_state[0][3] = true
```

| SendGpoMask(ch, matrix) | int, int | Wysyla maske GPO do klienta ch |
