# RPC-006: JACK Audio Port Management

## Kontekst biznesowy

JACK (JACK Audio Connection Kit) to profesjonalny system audio dla Linux umozliwiajacy routing audio miedzy aplikacjami z niskim opoznieniem. ripcd obsluguje trzy komendy RML do zarzadzania polaczeniami portow JACK: JC (connect), JD (disconnect), JZ (disconnect all). Cala feature jest warunkowa -- kompilowana tylko jesli ifdef JACK jest zdefiniowany. Jesli JACK nie jest dostepny, komendy zwracaja blad.

## Aktorzy

| Aktor | Rola w tej feature |
|-------|-------------------|
| Operator | Steruje polaczeniami JACK via RML |
| JACK server | Serwer audio zarzadzajacy portami |

## Granica funkcjonalnosci

```
IN SCOPE:
  - JACK client initialization (jack_client_open, startJackData timer)
  - RML komendy: JC (connect), JD (disconnect), JZ (disconnect all)
  - ripcd_jack_client handle
  - ripcd_start_jack_timer (5s delay na starcie)
OUT OF SCOPE:
  - Audio playback/recording -> patrz CAE
  - Inne komendy RML -> patrz RPC-003
```

## Use Cases

| ID | Aktor | Akcja | Efekt | Priorytet |
|----|-------|-------|-------|-----------|
| UC-1 | Operator | RML JC port_in port_out | Polacz porty JACK | MUST |
| UC-2 | Operator | RML JD port_in port_out | Rozlacz porty JACK | MUST |
| UC-3 | Operator | RML JZ | Rozlacz WSZYSTKIE polaczenia | SHOULD |
| UC-4 | System | Startup | Inicjalizacja klienta JACK (po 5s delay) | MUST |

## Reguly biznesowe (Gherkin)

```gherkin
Rule: JACK connect

  Scenario: Polaczenie portow JACK
    Given JACK client aktywny (ripcd_jack_client != NULL)
    And   RML "JC port_in port_out" (2 argumenty)
    When  RunLocalMacros przetwarza JC
    Then  jack_connect(client, port_out, port_in)
    And   log do syslog
    And   acknowledge(true)
  # Zrodlo: local_macros.cpp:417-419 | Pewnosc: potwierdzone

  Scenario: EEXIST ignorowane
    Given porty juz polaczone
    When  jack_connect zwraca EEXIST
    Then  NIE loguje warning (ignoruje)
  # Zrodlo: local_macros.cpp:426 | Pewnosc: potwierdzone

Rule: JACK disconnect all

  Scenario: Rozlacz wszystkie polaczenia
    Given JACK client aktywny
    When  RML "JZ" odebrane (0 argumentow)
    Then  iteruje wszystkie output porty (jack_get_ports)
    And   dla kazdego: pobierz polaczenia (jack_port_get_all_connections)
    And   rozlacz kazde polaczenie (jack_disconnect)
  # Zrodlo: local_macros.cpp:507-535 | Pewnosc: potwierdzone

Rule: JACK niedostepny

  Scenario: JACK nie skompilowany
    Given ifdef JACK nie zdefiniowany
    When  RML JC/JD/JZ odebrane
    Then  acknowledge(false)
  # Zrodlo: local_macros.cpp:446-451 | Pewnosc: potwierdzone

  Scenario: JACK client NULL (nie udalo sie polaczyc)
    Given ripcd_jack_client == NULL
    When  RML JC odebrane
    Then  acknowledge(false)
  # Zrodlo: local_macros.cpp:435-440 | Pewnosc: potwierdzone

Rule: JACK startup delay

  Scenario: Inicjalizacja klienta JACK
    Given ripcd startuje z ifdef JACK
    When  ripcd_start_jack_timer wygasa (po 5000ms)
    Then  startJackData() probuje jack_client_open
    And   jesli sukces: ripcd_jack_client = handle
    And   jesli blad: ripcd_jack_client = NULL
  # Zrodlo: ripcd.cpp:204-205 | Pewnosc: potwierdzone
```
