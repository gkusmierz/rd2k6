# LIB-001: Domain Model Core (Cart, Cut, Group)

## Kontekst biznesowy

Fundament calego systemu — encje Cart (kontener audio), Cut (segment audio) i Group (grupa cartow) sa uzywane przez kazdy inny modul. Cart jest podstawowa jednostka biblioteki audio — moze zawierac wiele cutow z rotation, metadanymi, kodami schedulera. Group definiuje zakresy numerow i reguly zarzadzania cartami.

## Aktorzy

| Aktor | Rola |
|-------|------|
| Operator | Tworzy, edytuje, przeszukuje carty i cuty |
| System | Wybiera cuty do odtworzenia (rotation), waliduje waznosc |
| Administrator | Konfiguruje grupy, zakresy numerow |

## Granica funkcjonalnosci

```
IN SCOPE:
  - Encja Cart: CRUD, metadane, rotation, validation
  - Encja Cut: CRUD, markery, dayparting, waznosc
  - Encja Group: CRUD, zakresy numerow, auto-assign
  - Encja SchedCode: etykiety schedulera
  - Encja WaveData: kontener metadanych audio

OUT OF SCOPE:
  - Audio playback (LIB-003)
  - Audio import/export (LIB-004)
  - UI dialogi (LIB-010)
  - Reporting (LIB-012)
```

## Reguly biznesowe (Gherkin)

```gherkin
Rule: Cut validity (dayparting, daty, dni tygodnia, evergreen)
  Scenario: Evergreen cut jest zawsze wazny
    Given cut z flaga evergreen
    When sprawdzana waznosc
    Then wazny

  Scenario: Cut poza zakresem dat
    Given cut z endDatetime < now
    When sprawdzana waznosc
    Then niewazny

Rule: Rotation (sekwencyjna/losowa, pomijanie niewaznych)
  Scenario: Sekwencyjna rotation
    Given cart z PlayOrder=Sequence, last=B
    When selectCut()
    Then wybrany C

Rule: Grupa kontroluje zakres numerow
  Scenario: Cart poza zakresem grupy
    Given grupa z enforceCartRange, range 100000-199999
    When tworzony cart 200001
    Then odrzucony

Rule: Timescaling 83.3%-125%
  Scenario: Cut poza zakresem timescale
    Given forcedLength > 125% dlugosci cuta
    When validateLengths()
    Then false
```
