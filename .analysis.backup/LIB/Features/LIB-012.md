# LIB-012: Reporting & Export

## Kontekst biznesowy

System raportowania generuje raporty z danych odtwarzania w 14 formatach. Raporty sa wymagane przez regulatora (FCC), firmy licencyjne (BMI, ASCAP), systemy traffic i wewnetrzna analityke. Kazdy format ma wlasna logike filtrowania i formatowania.

## Aktorzy

| Aktor | Rola |
|-------|------|
| Administrator | Konfiguruje raporty, generuje |
| System | Automatyczne generowanie po zakonczeniu dnia |

## Granica funkcjonalnosci

```
IN SCOPE:
  - RDReport: CRUD, generowanie raportu
  - 14 eksporterow (export_*.cpp): DeltaFlex, TextLog, BMI EMR,
    Technical, SoundEx, NPR SoundEx, RadioTraffic, MusicClassical,
    MusicPlayout, MusicSummary, SpinCount, CutLog, ResultsReport

OUT OF SCOPE:
  - UI administracyjne (artefakt ADM)
```

## Reguly biznesowe (Gherkin)

```gherkin
Rule: Raport filtruje dane wg konfiguracji
  Scenario: Filtrowanie po grupach
    Given raport z filterGroups=["Music", "Spots"]
    When generateReport(date)
    Then raport zawiera tylko odtworzenia z tych grup

Rule: Raport obsluguje on-air flag
  Scenario: Filtrowanie po on-air
    Given raport z filterOnairFlag=true
    When generateReport(date)
    Then raport zawiera tylko odtworzenia gdy stacja byla on-air
```
