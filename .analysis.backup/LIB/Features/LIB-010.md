# LIB-010: Shared UI Dialogs & Widgets

## Kontekst biznesowy

Zbiory wielokrotnie uzywanych dialogow (19 RDDialog subklas) i widgetow (17 QWidget subklas) stanowiacych wspolna warstwe UI dla wszystkich aplikacji Rivendell. Obejmuje dialogi wyboru (cart, cut, log, grupa, data), edycji (audio markers, metadane, ustawienia), widgety metrowe, suwaki, listy, pola edycji.

## Aktorzy

| Aktor | Rola |
|-------|------|
| Operator | Uzywa dialogow i widgetow w codziennej pracy |

## Granica funkcjonalnosci

```
IN SCOPE:
  - Dialogi: RDCartDialog, RDCutDialog, RDEditAudio, RDImportAudio,
    RDAddCart, RDAddLog, RDWaveDataDialog, RDExportSettingsDialog,

    RDListGroups, RDListSvcs, RDDateDialog, RDGetPasswd, RDPasswd,
    RDGetAth, RDEditPanelName, RDBusyDialog, RDCueEditDialog
  - Widgety: RDPushButton, RDTransportButton, RDComboBox, RDLineEdit,
    RDSlider, RDPlayMeter, RDSegMeter, RDStereoMeter, RDBusyBar,
    RDTimeEdit, RDDatePicker, RDLogFilter, RDEmptyCart, RDCardSelector,
    RDGpioSelector, RDListSelector, RDImagePickerBox, RDImagePickerModel,
    RDMarkerEdit, RDMarkerBar, RDWavePainter
  - Klasy bazowe: RDDialog, RDWidget, RDFrame, RDFontEngine

OUT OF SCOPE:
  - Logika biznesowa dialogow (nalazy do odpowiednich FEAT)
```

## Reguly biznesowe (Gherkin)

```gherkin
Rule: RDCartDialog obsluguje filtrowanie
  Scenario: Filtrowanie po grupie
    Given operator wybiera grupe "Music"
    When lista jest odswiezona
    Then widoczne sa tylko carty z grupy Music

Rule: RDEditAudio obsluguje 7 typow markerow
  Scenario: Ustawienie markera
    Given kursor na pozycji 5000ms
    When operator klika "Segue Start"
    Then marker Segue Start ustawiony na 5000ms

Rule: RDAddCart waliduje dane
  Scenario: Brak tytulu
    Given operator nie podal tytulu
    When klika OK
    Then komunikat bledu "Title required"
```
