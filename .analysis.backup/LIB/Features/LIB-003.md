# LIB-003: Audio Playback Engine

## Kontekst biznesowy

Silnik odtwarzania logu (RDLogPlay) i deck odtwarzania (RDPlayDeck) sa sercem automatyki radiowej. Zarzadzaja kolejka odtwarzania, automatycznymi przejsciami (segue/play/stop), hard time events, grace timerami, timescalingiem i komunikacja z serwerem audio. Obsluguja do 7 jednoczesnych deckow.

## Aktorzy

| Aktor | Rola |
|-------|------|
| Operator | Uruchamia/zatrzymuje/pauzuje odtwarzanie, ustawia next |
| System | Automatyczne segue, hard time start, grace timer |

## Granica funkcjonalnosci

```
IN SCOPE:
  - RDLogPlay: silnik odtwarzania logu
  - RDPlayDeck: pojedynczy deck odtwarzania
  - RDSimplePlayer: prosty odtwarzacz
  - Timescaling, fade, segue, hook, duck
  - PAD (Program Associated Data) output

OUT OF SCOPE:
  - Komunikacja z caed (LIB-007)
  - Panel dzwiekowy (LIB-005)
  - Rendering logu do pliku (LIB-004)
```

## Reguly biznesowe (Gherkin)

```gherkin
Rule: Automatyczne przejscia (segue/play/stop)
  Scenario: Segue
    Given element A z segue point, element B z transType=Segue
    When A dojdzie do segue point
    Then B startuje automatycznie, overlap

  Scenario: Hard time event
    Given element z timeType=Hard, startTime=14:00
    When zegar = 14:00
    Then element startuje automatycznie

Rule: PlayDeck state machine (Idle->Playing->Paused->Stopping->Stopped)
  Scenario: Natural end
    Given deck Playing
    When audio konczy sie naturalnie
    Then deck przechodzi do Stopped
```
