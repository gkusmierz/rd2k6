# LIB-008: User Management & Authentication

## Kontekst biznesowy

Zarzadzanie uzytkownikami obejmuje uwierzytelnianie (lokalne haslo lub PAM), autoryzacje (20+ flag uprawnien), zarzadzanie grupami i serwisami uzytkownika, tickety do web API.

## Aktorzy

| Aktor | Rola |
|-------|------|
| Administrator | Tworzy/edytuje uzytkownikow, nadaje uprawnienia |
| Operator | Loguje sie, zmienia haslo |
| System | Waliduje tickety web API |

## Granica funkcjonalnosci

```
IN SCOPE:
  - RDUser: CRUD, uwierzytelnianie, autoryzacja
  - Uprawnienia: 20+ flag (adminConfig, createCarts, editAudio, etc.)
  - Tickety: tworzenie, walidacja
  - RDGetPasswd, RDPasswd, RDGetAth: dialogi

OUT OF SCOPE:
  - UI administracyjne (artefakt ADM)
```

## Reguly biznesowe (Gherkin)

```gherkin
Rule: Dwie metody uwierzytelniania
  Scenario: Lokalne haslo
    Given user z localAuthentication=true
    When checkPassword("correct")
    Then true

  Scenario: PAM
    Given user z localAuthentication=false, pamService="rivendell"
    When checkPassword(password)
    Then weryfikacja przez PAM

Rule: Web user wymaga flagi enableWeb
  Scenario: Web login bez flagi
    Given user z enableWeb=false
    When logowanie przez web API
    Then odmowa dostepu
```
