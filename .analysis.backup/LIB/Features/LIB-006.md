# LIB-006: Program Generation

## Kontekst biznesowy

Generowanie programu radiowego odbywa sie przez hierarchie: Service -> Grid (siatka 7x24) -> Clock (zegar godzinowy) -> Event (zdarzenie). System automatycznie generuje logi (playlisty) na podstawie siatki programu, importuje dane z systemow music/traffic, i linkuje importy do logow.

## Aktorzy

| Aktor | Rola |
|-------|------|
| System | Generuje logi automatycznie |
| Administrator | Konfiguruje siatke, zegary, zdarzenia |

## Granica funkcjonalnosci

```
IN SCOPE:
  - RDSvc: serwis — generowanie logow, import music/traffic
  - RDClock: zegar — definiuje 1 godzine programu
  - RDEvent: zdarzenie — reguly importu, przejscia
  - RDEventLine: pozycja zdarzenia w zegarze
  - RDGrid: siatka 168 slotow (7 dni x 24h)
  - RDSchedCartList, RDSchedRulesList: scheduling

OUT OF SCOPE:
  - Odtwarzanie logu (LIB-003)
  - UI konfiguracji (aplikacja rdadmin)
```

## Reguly biznesowe (Gherkin)

```gherkin
Rule: Generowanie logu z siatki
  Scenario: Pelna generacja
    Given serwis z kompletna siatka
    When generateLog(date)
    Then log jest tworzony z odpowiednimi zdarzeniami

Rule: Import music/traffic
  Scenario: Import pliku music
    Given event z importSource=Music
    And plik importu dla daty
    When import()
    Then dane music sa wstawione jako linki w logu

Rule: Scheduler codes zapobiegaja powtorzeniom
  Scenario: Separacja artystow
    Given regula artystSep=60 (60 min separacji)
    When scheduler wybiera carta
    Then ten sam artysta nie moze pojawic sie w ciagu 60 min
```
