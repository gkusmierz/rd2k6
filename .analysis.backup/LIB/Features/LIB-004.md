# LIB-004: Audio Conversion & Import/Export

## Kontekst biznesowy

Konwersja audio jest kluczowa dla importu/eksportu plikow miedzy roznymi formatami (WAV, MP3, OGG, FLAC, M4A). Pipeline 3-etapowy (decode->resample->encode) obsluguje timescaling i normalizacje. RDAudioImport/Export laczy sie z web API do transferu plikow. RDRenderer miesza log do jednego pliku audio.

## Aktorzy

| Aktor | Rola |
|-------|------|
| Operator | Importuje/eksportuje pliki audio |
| System | Konwertuje audio przy publikacji podcastow, renderowaniu |

## Granica funkcjonalnosci

```
IN SCOPE:
  - RDAudioConvert: pipeline 3-etapowy
  - RDAudioExport/Import: operacje na cutach
  - RDRenderer: rendering logu do pliku
  - RDWaveFile: parsowanie plikow audio
  - RDTrimAudio, RDRehash, RDPeaksExport
  - RDAudioSettings, RDAudioInfo, RDAudioStore

OUT OF SCOPE:
  - UI dialog importu (LIB-010)
  - Playback (LIB-003)
```

## Reguly biznesowe (Gherkin)

```gherkin
Rule: Pipeline 3-etapowy
  Scenario: Konwersja MP3 do WAV
    Given plik zrodlowy MP3
    When convert()
    Then Stage1: dekodowanie MP3 do PCM
    And Stage2: resampling (jesli potrzebny)
    And Stage3: zapis jako WAV PCM16

Rule: Timescaling w zakresie 83.3%-125%
  Scenario: SpeedRatio poza zakresem
    Given speedRatio=1.30
    When convert()
    Then blad — poza zakresem
```
